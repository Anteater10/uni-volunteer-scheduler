The three analytics endpoints that currently raise 501 need to be reimplemented using the `Volunteer` model instead of the old `User` model. The `Signup` model now has a `volunteer_id` FK and a `volunteer` relationship (from Phase 08).

2. **Update `schemas.py` — `NoShowRateRow`:**
   - Change `user_id: UUID` to `volunteer_id: UUID`
   - Change `name: str` to `volunteer_name: str` (same reasoning)

3. **Reimplement `GET /admin/analytics/volunteer-hours` in admin.py:**
   - Remove the 501 raise and the Phase 12 TODO comments.
   - Query: join `Signup` -> `Slot` -> `Event`, group by `signup.volunteer_id`.
   - For each volunteer, calculate hours = sum of `(slot.end_time - slot.start_time)` for signups with status `attended`.
   - Count distinct events.
   - Apply `from_date` / `to_date` filters on `Event.start_date` (same pattern as the working `attendance_rates` endpoint).
   - Join `Volunteer` to get `first_name + last_name`.
   - Return list of `VolunteerHoursRow`.
   - Add audit log: `log_action(db, admin_user, "admin_analytics_volunteer_hours", "Analytics", None)`.

4. **Reimplement `GET /admin/analytics/no-show-rates` in admin.py:**
   - Remove the 501 raise.
   - Query: group signups by `volunteer_id`, count `attended` vs `no_show` statuses.
   - Rate = `no_show / (attended + no_show)` for each volunteer (skip volunteers with zero denominator).
   - Apply date filters via `Signup` -> `Slot` -> `Event.start_date`.
   - Return list of `NoShowRateRow`.

5. **Reimplement `GET /admin/analytics/volunteer-hours.csv` in admin.py:**
   - Remove the 501 raise.
   - Reuse the same query logic as `volunteer-hours` JSON endpoint.
   - Output CSV with columns: `volunteer_name, email, hours, events`.
   - Use `io.StringIO` + `csv.writer` (same pattern as the existing `export_event_attendance_csv` endpoint at line ~848).
   - Return `Response(content=..., media_type="text/csv", headers=...)`.

6. **Fix CCPA export signups stub in admin.py (`ccpa_export` function):**
   - At line ~952 where `signups_data = []`, replace with:
     - Look up the User's email: `user.email`.
     - Query `Volunteer` by that email: `vol = db.query(models.Volunteer).filter(models.Volunteer.email == user.email).first()`.
     - If volunteer found, query their signups: `vol.signups` or `db.query(models.Signup).filter(models.Signup.volunteer_id == vol.id).all()`.

   - NOTE: This approach links User to Volunteer by email match. This is the simplest correct approach since there's no direct FK between User and Volunteer.
