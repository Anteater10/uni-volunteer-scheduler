<!-- Backend API contracts the frontend consumes -->

GET /admin/imports -> list of CsvImportRead:
```json
[{
  "id": "uuid",
  "filename": "spring2026.csv",
  "status": "pending" | "processing" | "ready" | "committed" | "failed",
  "result_payload": {
    "raw_csv": "...",
    "rows": [{ "index": 0, "status": "ok"|"low_confidence"|"conflict", "normalized": {...}, "warnings": [...], "original": {...} }],
    "summary": { "to_create": 10, "to_review": 2, "conflicts": 1, "total": 13 }
  },
  "error_message": null | "string",
  "created_at": "iso8601"
}]
```

GET /admin/imports/{id} -> single CsvImportRead (same shape as above)

PATCH /admin/imports/{id}/rows/{index} -> updated result_payload
  Body: { "module_slug": "corrected-slug", ... }
  Effect: row status changes from "low_confidence" to "ok", summary recalculated

POST /admin/imports/{id}/commit -> { "created_count": N, "events": [...] }
  Rejects if any low_confidence rows remain (400)

POST /admin/imports/{id}/retry -> { "status": "retrying", "import_id": "..." }
  Only works on failed imports

**2. Rewrite ImportsSection.jsx:**

Keep the existing upload mutation and imports list query. Add:

**a) Polling on the list query when any import is pending/processing:**
```jsx
const importsQ = useQuery({
  queryKey: ["adminImports"],
  queryFn: () => api.admin.imports.list(),
  refetchInterval: (query) => {
    const data = query.state.data;
    if (!data) return false;
    const hasActive = data.some(
      (imp) => imp.status === "pending" || imp.status === "processing"
    );
    return hasActive ? 2000 : false;
  },
});
```

**b) Selected import detail state:**

When admin clicks a row in the table (any row with status `ready` or `processing` or `pending`), set `selectedImportId` state. Show a detail panel below the table.

**c) Preview table in the detail panel:**

When selectedImport is `ready` and has `result_payload.rows`:
- Show a summary banner at the top:
  ```
  "12 events ready to create, 2 need your review, 1 has a scheduling conflict"
  ```
  Use green/yellow/red Chip components for the counts.

- Render a table with columns: **#** (row index+1) / **Module** (normalized.module_slug) / **Date** (normalized.start_at formatted) / **Location** (normalized.location) / **Status** (Chip: green "Ready", yellow "Needs Review", red "Conflict") / **Warnings** (list of warning strings)

- Row background colors:
  - `ok` rows: no special background
  - `low_confidence` rows: `bg-yellow-50 border-l-4 border-yellow-400`
  - `conflict` rows: `bg-red-50 border-l-4 border-red-400`

**d) Inline row editing for low-confidence rows:**

When admin clicks a `low_confidence` row, expand an inline edit form below it (or open a small modal) showing editable fields: module_slug (text input), location (text input), capacity (number input). On save, call `api.admin.imports.updateRow(importId, rowIndex, { module_slug, location, capacity })`. On success, invalidate the `adminImports` query. The backend automatically changes the row status to `ok` and recalculates the summary.

**e) Commit button gated on summary:**

The Commit button in the detail panel is disabled when `summary.to_review > 0`. Show tooltip text: "Resolve all flagged rows before committing." When `to_review === 0`, the button is enabled. On click, show the existing commit confirmation modal but with updated copy that includes the summary:
```
"This will create {to_create} events. {conflicts} conflicting rows will be skipped. This cannot be undone."
```

**f) Processing indicator:**

When selectedImport status is `pending` or `processing`, show a pulsing "Processing your CSV..." message with a Skeleton animation instead of the preview table.

**g) Error messages in plain English:**

Replace raw `imp.error_message` display with a helper:
```jsx
function humanizeError(msg) {
  if (!msg) return null;
  if (msg.includes("AuthenticationError") || msg.includes("API key"))
    return "The AI service could not authenticate. Ask your admin to check the API key.";
  if (msg.includes("cost") && msg.includes("ceiling"))
    return "This CSV is too large to process. Try splitting it into smaller files.";
  if (msg.includes("timeout") || msg.includes("Timeout"))
    return "Processing took too long. Try again or use a smaller file.";
  return "Something went wrong during processing. Click Re-run to try again.";
}
```

**h) Explainer sentence at top of page (per D-18 cross-cutting rule):**
