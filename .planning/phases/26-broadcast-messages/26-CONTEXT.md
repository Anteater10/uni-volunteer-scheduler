# Phase 26: Broadcast messages — Context

**Gathered:** 2026-04-17

<domain>
## Phase Boundary

Organizer/admin → email all confirmed signups for an event in one shot ("parking moved to Lot 22"). Rate-limited, audited, markdown body. Reuses email infra from Phase 24.

</domain>

<decisions>
## Implementation Decisions

### Who can broadcast
- Admin on any event.
- Organizer on events they own (or for v1.3 single-tenant simplicity: any organizer on any event — we're not modeling event ownership yet).

### Recipients
- `confirmed` + `checked_in` + `attended` signups receive. Waitlisted + cancelled + no_show do NOT receive. (Why: waitlisted don't yet have a spot; cancelled opted out; no-show info is post-event noise.)

### Rate limit
- 5 broadcasts / hour / event. Enforced at router level. New table `broadcast_rate_limits` OR reuse `sent_notifications` table — decision: use a simple in-memory or redis counter keyed by `(event_id, hour_bucket)`. Use existing redis connection from celery. 429 on exceed with `Retry-After` header.

### Content
- Subject: up to 200 chars.
- Body: markdown, rendered to HTML via `markdown` pip package (already a dep? check; add if missing).
- Plain-text version auto-generated by stripping HTML tags.
- Auto-appended footer: event title + start time (PT) + venue + unsubscribe link (links to manage page for that signup's volunteer).

### Idempotency
- Each broadcast gets a UUID. `sent_notifications` rows inserted with `kind=f"broadcast_{broadcast_id}"` per signup — dedupe logic prevents double-send if the request is retried.

### Audit log
- One row per broadcast: actor, event, broadcast_id, subject, recipient_count.

### API
- `POST /events/{id}/broadcast`:
  - Body: `{subject: str, body_markdown: str}`.
  - Auth: admin OR organizer role.
  - Returns `{broadcast_id, recipient_count, sent_at}`.
- `GET /events/{id}/broadcasts` — list recent broadcasts for admin preview (past 30 days).

### Frontend
- `frontend/src/components/BroadcastModal.jsx` (new):
  - Subject input (200 char counter).
  - Markdown textarea (basic preview pane via `marked` or `react-markdown`).
  - Recipient count preview (fetched from API).
  - Send button with confirmation dialog.
- Wired from `AdminEventPage.jsx` "Message volunteers" button AND organizer roster page.

### Tests
- `backend/tests/test_broadcast_service.py`: success path (N recipients), rate limit 429, markdown→HTML+plain, audit row written, idempotency dedup.
- Frontend: modal state + send action + recipient count display.

</decisions>
