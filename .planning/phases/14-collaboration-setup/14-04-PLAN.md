<objective>
Run the one-day parallel test mandated by ROADMAP Phase 14 Success Criterion #5 ("Both devs run a one-day parallel test: each opens a Claude Code session on a different pillar branch, completes one trivial change, merges to main without conflict"). Each dev makes a deliberately conflict-inducing trivial change to the same file (`docs/weeknotes.md`) on their own role branch, then merges in sequence to main, resolving the resulting conflict by keeping both lines. The point is to prove the conflict-resolution playbook from `docs/COLLABORATION.md` actually works in practice, not just on paper.

Purpose: This is the only verification in Phase 14 that exercises the workflow end-to-end with both real devs. Plans 01-03 produce documents and branches; Plan 04 proves the documents and branches actually let two humans on two machines work in parallel and reconcile their changes. If this test fails, Phase 14 is incomplete regardless of how good the docs look.

Output: Two trivial commits (one Andy, one Hung) on different role branches, merged to main with one resolved conflict, plus a short documented record at `.planning/phases/14-collaboration-setup/14-PARALLEL-TEST.md`.

This plan is deliberately `autonomous: false` because Hung's commits cannot be made by the executor — Hung must run them on his own machine. The executor handles Andy's side and prints clear handoff instructions for Hung.
</objective>
