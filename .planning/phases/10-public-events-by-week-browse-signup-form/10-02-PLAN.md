**Page structure (mobile-first per D-06, 375px base):**

1. **Week navigator header:**
   - Left arrow button (ChevronLeft from lucide-react), "Week N" label (from formatWeekLabel), right arrow button (ChevronRight)
   - Below: "This week" Button (variant="secondary", small) that resets to getCurrentWeek result
   - Touch targets >= 44px (min-h-11 min-w-11 on arrow buttons)
   - Use useSearchParams from react-router-dom to sync quarter/year/week_number to URL query params per D-10: `?quarter=spring&year=2026&week=3`

2. **Data fetching:**
   - On mount: call api.public.getCurrentWeek() via useQuery to get default week. Use this as initial values if no URL params present.
   - Main query: useQuery with queryKey ["publicEvents", quarter, year, weekNumber], calls api.public.listEvents({quarter, year, week_number: weekNumber}). Only enabled when all 3 params are set.
   - On 429 error: show toast.error("Please wait a moment and try again") per D-08.

3. **Event cards:**
   - Group events by school (reduce into {school: events[]} object)
   - Each school section: small uppercase heading (text-sm font-medium uppercase tracking-wide text-[var(--color-fg-muted)])
   - Each event: Card component containing title, date range (formatted), slot count summary (e.g., "3 orientation + 5 period slots"), capacity indicator
   - Card is a Link to /events/{event.id} (use react-router-dom Link)

4. **States:**
   - Loading: 3 Skeleton cards (h-24 rounded-xl)
   - Empty: EmptyState with title="No events this week" and body="Try browsing a different week."
   - Error: EmptyState with error message and Retry button

**Route wiring in App.jsx per D-05:**
- Change the `/events` route import from EventsPage to EventsBrowsePage (lazy import from pages/public/EventsBrowsePage)
- Change the `/events/:eventId` route import from EventDetailPage to EventDetailPage from pages/public/EventDetailPage (will be created in Plan 03; for now, keep the old import but add a comment marking it for Plan 03 replacement)
- Keep old page files intact (Phase 12 removes them)
- Do NOT add any ProtectedRoute wrapper around these routes -- they must remain public per D-05 and REQ-10-07

IMPORTANT: Do NOT call useAuth() as a render gate. The page must render for logged-out users.
