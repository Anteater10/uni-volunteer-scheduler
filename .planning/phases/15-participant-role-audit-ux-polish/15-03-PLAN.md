<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend API → page render | Event titles, descriptions, portal name/events come from server JSON |
| User → query refetch click | onClick on "Try again" calls react-query refetch |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-03-01 | Tampering | Server-rendered event/portal text in JSX | mitigate | React default-escapes string children. No `dangerouslySetInnerHTML` introduced. ErrorState body is a hardcoded literal (not user-supplied). Empty-state body is a hardcoded literal. PartialPage portal name/tagline rendered via plain JSX text — safe. |
| T-15-03-02 | Information Disclosure | `eventsQ.error?.message` previously rendered to user | mitigate | OLD code rendered raw error messages (potentially leaking backend internals). NEW code replaces with hardcoded UI-SPEC copy. Removes leakage path. |
| T-15-03-03 | Tampering | Refetch onClick handler | accept | React-query refetch is idempotent and goes through existing api.public.* wrapper (D-14 — api.js read-only). No new attack surface. |
| T-15-03-04 | Denial of Service | "Try again" button spammed | accept | react-query has built-in deduplication + retry caps. UI-SPEC does not require explicit rate-limit on the button. |
| T-15-03-05 | Spoofing | aria-live="polite" announcement | accept | Standard ARIA pattern. Cannot be abused — assistive tech only announces visible content. |
</threat_model>
