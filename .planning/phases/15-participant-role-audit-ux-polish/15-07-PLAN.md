    **Implementation pattern** (add near top of file, inside `test.describe` wrapper if one exists):

    ```javascript
    // PART-02 — no console errors / pageerrors on any public route during golden path
    // Allow-list: empty today. Add entries ONLY with explicit justification comment.
    const ALLOWED_CONSOLE_PATTERNS = [
      // e.g. /Download the React DevTools/ (dev-only noise) — uncomment if it surfaces in CI
    ]

    test.beforeEach(async ({ page }, testInfo) => {
      testInfo.errors = []
      page.on('pageerror', (err) => {
        testInfo.errors.push(`pageerror: ${err.message}`)
      })
      page.on('console', (msg) => {
        if (msg.type() !== 'error') return
        const text = msg.text()
        if (ALLOWED_CONSOLE_PATTERNS.some((re) => re.test(text))) return
        testInfo.errors.push(`console.error: ${text}`)
      })
    })

    test.afterEach(async ({}, testInfo) => {
      const errors = testInfo.errors || []
      if (errors.length > 0) {
        throw new Error(
          `PART-02 violation — ${errors.length} error(s) captured during "${testInfo.title}":\n${errors.join('\n')}`,
        )
      }
    })
    ```

    **Key constraint:** Use `testInfo.errors` (per-test bag) not a module-level array, so parallel tests don't bleed errors across each other. Playwright runs tests in parallel within a project when `fullyParallel: true` (already configured).

    **Do NOT:**
    - Add new test cases — just wire listeners into existing ones
    - Baseline existing errors silently by populating ALLOWED_CONSOLE_PATTERNS without a code comment explaining which real error is being allowed and why
    - Disable the assertion for a specific test (if a test genuinely needs a console warning, add a scoped `ALLOWED_CONSOLE_PATTERNS` local to that test)
  </action>
