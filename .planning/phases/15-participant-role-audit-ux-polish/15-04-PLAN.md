    function isValidPhone(raw) {
      if (!raw) return false

      // E.164: +[country][number], total 8–15 digits
      if (/^\+[1-9]\d{7,14}$/.test(trimmed)) return true
      // US: strip non-digits, require exactly 10 digits
      const digitsOnly = trimmed.replace(/\D/g, '')
      if (digitsOnly.length === 10) return true
      // US with leading 1: 11 digits starting with 1
      if (digitsOnly.length === 11 && digitsOnly.startsWith('1')) return true
      return false
    }

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend → event/slot JSON → page render | Event title, description, school, slot location, times come from server |
| User form input → validateIdentity → POST /public/signups | Name, email, phone (user-supplied, client-validated, server-authoritative) |
| .ics filename arg → browser download UI | Constructed from `event.slug || event.id` + `event.start_date` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-04-01 | Tampering | Event description/title rendered in JSX | mitigate | React default-escapes. No `dangerouslySetInnerHTML` introduced. Confirmed by grep after edits. |
| T-15-04-02 | Tampering | .ics DESCRIPTION + SUMMARY via downloadIcs | mitigate | Plan 02's `escapeText()` escapes RFC 5545 special characters. This plan only calls downloadIcs — does not bypass escaping. Unit-tested in calendar.test.js. |
| T-15-04-03 | Tampering | .ics filename arg | mitigate | Filename is `scitrek-{slug|id}-{date}.ics`. `event.slug` is server-generated (kebab-case lowercase, no path separators); `event.id` is integer; `event.start_date` is ISO date. No user free-text flows into filename. Browser download UI sanitizes further. |
| T-15-04-04 | Input Validation | validateIdentity E.164 + US phone | mitigate | Client validation tightened per PART-05. Server remains authoritative (D-14 — no backend change). isValidPhone rejects invalid strings before submit, improving UX; attackers cannot bypass server-side Pydantic validation. |
| T-15-04-05 | Information Disclosure | `eventQ.error?.message` removal | mitigate | Old code leaked backend error details into user-facing UI. New UI-SPEC copy is a fixed string. Closes disclosure channel. |
| T-15-04-06 | Denial of Service | Add-to-calendar button spam | accept | downloadIcs is a synchronous browser operation. No backend call. Spam creates local Blob objects (garbage-collected). Low risk. |
| T-15-04-07 | Spoofing | Toast message "Calendar file saved" | accept | Toast is shown after synchronous downloadIcs invocation. If the browser blocks the download (e.g. permissions denied), the toast may over-promise. UI-SPEC copy is "saved" not "imported"; acceptable. |
</threat_model>
