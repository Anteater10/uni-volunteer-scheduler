## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → npm registry | `playwright install --with-deps` downloads browser binaries from Microsoft's CDN. Supply-chain integrity is delegated to npm + Microsoft. |
| ErrorState rendering → user viewport | React JSX; Title and body props are rendered via text children. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-15-01-01 | Tampering | ErrorState title/body props | mitigate | React default-escapes string children via JSX. No `dangerouslySetInnerHTML` anywhere in ErrorState. Body rendered inside `<p>{body}</p>` — safe. Pattern PATTERNS.md §ErrorState confirms no HTML injection surface. |
| T-15-01-02 | Denial of Service | Playwright CI install | accept | Three-engine install triples CI install time (~3x per RESEARCH.md A4). Accepted trade-off for PART-14 cross-browser coverage. Sharding deferred per A4. |
| T-15-01-03 | Information Disclosure | playwright.config.js trace/video | accept | `trace: 'on-first-retry'` + `video: 'retain-on-failure'` preserve failure artifacts. Artifacts contain seeded test data only (no real PII — backend runs with test DB). Unchanged from existing config. |
| T-15-01-04 | Tampering | ErrorState icon prop | mitigate | `icon` prop accepts a React component, not a string. JSX renders the component — cannot inject raw HTML. Default (AlertTriangle) hardcoded. |
| T-15-01-05 | Spoofing | CI `playwright install` download | accept | Browser binaries are npm-tag-locked to `@playwright/test@1.59.1` per package.json; npm integrity hashes enforced by lockfile. Standard npm supply-chain risk — no new exposure. |
</threat_model>
