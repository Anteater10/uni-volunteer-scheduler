**Preconditions section** — numbered steps, each a literal copy-pasteable
command:
1. Fresh docker stack: `docker compose down -v` then
   `docker compose up -d db redis` then `docker compose run --rm migrate`
   then `docker compose up -d backend celery_worker celery_beat`.
2. Seed the E2E data: `EXPOSE_TOKENS_FOR_TESTING=1 python3 backend/tests/fixtures/seed_e2e.py`.
3. Frontend dev: `cd frontend && npm run dev`.

5. Open three browser tabs / windows:
   - Admin (desktop, 1280×800 viewport): logged in as
     admin@e2e.example.com / Admin!2345
   - Organizer (phone-sized, 375×812 viewport via devtools or a real
     phone on same LAN): logged in as organizer@e2e.example.com /
     Organizer!2345
   - Participant (incognito, 375×812): clean / no cookies.

**Section 1 — Participant flow (phone, 375px)** — 10 checkboxes covering:
- `/events` loads, shows week nav, seeded event visible, no console errors
- Open event detail — slots grouped by orientation / period with
  capacity + filled counts
- Sign up for a period slot — orientation warning modal fires for a new
  email with no prior attendance, suppressed when using the seeded
  attended_volunteer_email
- Form validation — invalid email + invalid phone both rejected with
  clear messages
- Submit signup → success card shown → confirmation email arrives in
  Mailpit within 15s
- Click magic link in Mailpit email → lands on `/signup/confirm` → shows
  confirmed banner
- `/signup/manage?token=...` shows the signup with per-row Cancel +
  cancel-all buttons
- Self check-in via `/check-in/:signupId` works inside the time window
- No horizontal scroll on any page at 375px; all tap targets ≥44px
- No stuck spinners; loading / empty / error states all render correctly

**Section 2 — Admin flow (desktop, 1280px)** — 10 checkboxes covering
every sidebar item per docs/ADMIN-AUDIT.md and Phase 16 success
criteria:
- Login → `/admin` Overview shows live stats (Users, Events, Slots,
  Signups, Confirmed signups) and Recent Activity feed
- `/admin/audit-logs` — pagination works, kind filter + actor filter +
  date range + keyword search all apply
- `/admin/users` — list loads, can create an organizer via invite, can
  deactivate a user, CCPA export link present
- `/admin/portals` — list loads, can open a portal detail
- `/admin/templates` — list with slug/name/capacity/duration, create
  form works, edit works, delete/archive works
- `/admin/imports` — upload a sample CSV, preview shows "N events will
  be created, M skipped", confirm commits atomically
- `/admin/exports` — volunteer hours + attendance + no-show CSVs all
  download with real data
- **`Overrides` tab is NOT present** in the sidebar (Phase 16
  retirement regression check)
- Every admin page shows loading / empty / error states correctly
- No console errors in DevTools across the full sweep

**Section 3 — Organizer flow (phone, 375px)** — 6 checkboxes:
- Login → lands on `/organizer` (phone-first dashboard, NOT
  `/admin/events`)
- Dashboard shows Today / Upcoming / Past tabs; tap switches tabs
- "Open roster" button on event card navigates to
  `/organizer/events/:id/roster`
- Roster shows confirmed signups with tap-friendly check-in rows
- Tapping a row flips status to "checked in" optimistically
- Organizer sidebar does NOT show Users, Audit Logs, or Exports (RBAC
  regression check)

**Section 4 — Cross-role loop** — 6 checkboxes, humans driving the three
tabs in one sitting (mirrors Scenario 1 from 20-01):
- Admin confirms the seed event exists in `/admin/events/:id`
- Participant (incognito tab) signs up for a period slot, confirms via
  Mailpit magic link
- Organizer (phone tab) sees the new signup appear in the roster within
  ~6s (poll) or after reload
- Organizer checks the participant in; row status flips to "checked in"
- Admin (desktop tab) navigates to `/admin/audit-logs`, filters by the
  participant's email, sees `signup.created` + `signup.checked_in`
  entries
- Participant (incognito tab) visits `/signup/manage?token=...` and sees
  the signup marked checked in

**Section 5 — Regressions to watch** — 6 bullets of known-gotcha checks:
- CSV import copy everywhere says "quarterly", not "yearly"
- No "student account" or "student register/login" copy anywhere
- All links use `/organizer/*`, none use bare `/organize/*` (except the
  single deliberate redirect)
- Magic link email actually arrives in Mailpit (confirms
  Mailpit-vs-SES/SendGrid mode is correct for dev)
- No failed network requests in any DevTools Network tab during the
  full sweep
- No console errors / warnings in any role

**Exit criteria + sign-off block:**
- Every box checked in one sitting
- No manual DB nudges required
- No failed requests in any Network tab
- Sign off line: `Smoke passed by: ________  Date: ________`
